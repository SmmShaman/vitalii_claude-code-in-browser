## How to Work (Generator-Evaluator Pattern)

Based on Anthropic's engineering article on harness design for long-running apps:

### Method
1. **Generator agent** makes changes to specific files
2. **Evaluator agent** opens the live site, takes screenshots or interacts via Playwright MCP
3. Evaluator scores against 4 criteria (see below)
4. If score < threshold, feedback goes back to generator
5. Repeat 5-15 iterations per area

### 4 Evaluation Criteria
Score each 1-5:

| Criterion | Question |
|-----------|----------|
| **Design Quality** | Does this feel like a coherent whole, not a collection of parts? |
| **Originality** | Are there distinctive choices, or does it look AI-generated? |
| **Craft** | Is typography, spacing, and color harmony precise and intentional? |
| **Functionality** | Does it work well — navigation, modals, search, responsive? |

**Target:** 4+ on each criterion before moving to next area.

### Practical Setup
- Run `npm run dev` → open http://localhost:3000
- After each change round, check the live result
- Use browser DevTools for responsive testing
- Focus on ONE area at a time (don't scatter changes)

## What NOT to Do
- Don't make technical-only changes (user wants VISIBLE results)
- Don't change all files at once (risk of breaking things)
- Don't skip visual verification (look at the live site)
- Don't use elastic/bounce easing (Impeccable says it's dated)
- Don't use same gradient on all buttons (create hierarchy)
- Don't kill processes without asking
