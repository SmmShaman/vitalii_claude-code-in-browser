# Інструкція з Деплою на Netlify

## Автоматичний деплой з GitHub

### Крок 1: Підготовка GitHub репозиторію

Репозиторій вже підготовлений і код завантажено на GitHub.

### Крок 2: Підключення до Netlify

1. **Зайдіть на Netlify:**
   - Відкрийте https://www.netlify.com/
   - Натисніть **"Sign up"** або **"Log in"**
   - Виберіть **"Sign up with GitHub"** (це дозволить Netlify отримати доступ до ваших репозиторіїв)

2. **Додайте новий сайт:**
   - На дашборді Netlify натисніть **"Add new site"** → **"Import an existing project"**
   - Виберіть **"Deploy with GitHub"**
   - Знайдіть репозиторій `vitalii_claude-code-in-browser`
   - Виберіть гілку: `claude/portfolio-bento-grid-011CURe2swFk5krQjcH5hZay`

3. **Налаштування білда:**
   Netlify автоматично визначить налаштування з файлу `netlify.toml`, але перевірте:
   - **Build command:** `npm run build`
   - **Publish directory:** `dist`
   - **Branch:** `claude/portfolio-bento-grid-011CURe2swFk5krQjcH5hZay`

4. **Змінні оточення (Environment Variables):**
   Якщо ви хочете використовувати контактну форму з Supabase:
   - Перейдіть в **Site settings** → **Environment variables**
   - Додайте:
     - `VITE_SUPABASE_URL` = ваш Supabase URL
     - `VITE_SUPABASE_ANON_KEY` = ваш Supabase Anon Key

5. **Натисніть "Deploy site"**

### Крок 3: Автоматичний деплой працює!

✅ Тепер кожного разу коли я вношу зміни і пушу в GitHub:
1. GitHub отримує нові зміни
2. Netlify автоматично виявляє зміни
3. Netlify запускає білд (`npm run build`)
4. Netlify деплоїть новий білд
5. Сайт оновлюється автоматично (зазвичай за 1-2 хвилини)

### Крок 4: Підключення власного домену

Коли все влаштує:

1. **В Netlify Dashboard:**
   - Виберіть ваш сайт
   - Перейдіть в **Domain settings** → **Add custom domain**
   - Введіть ваш домен (наприклад, `vitaliiberbeha.com`)

2. **Налаштуйте DNS у вашого провайдера:**

   **Варіант А: Використання Netlify DNS (рекомендовано):**
   - Netlify покаже nameservers
   - Додайте їх у вашого domain registrar

   **Варіант Б: Власний DNS:**
   - Додайте A record: `104.198.14.52`
   - Або CNAME record: `ваш-сайт.netlify.app`

3. **SSL/HTTPS:**
   - Netlify автоматично додасть безкоштовний SSL сертифікат (Let's Encrypt)
   - Це займе 1-24 години після підключення домену

### Workflow для роботи:

```
┌─────────────────────────────────────────────────────┐
│  Claude Code → Робить зміни в коді                  │
└──────────────────┬──────────────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────────────┐
│  Ви → git pull && git push до GitHub                │
└──────────────────┬──────────────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────────────┐
│  GitHub → Отримує зміни                             │
└──────────────────┬──────────────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────────────┐
│  Netlify → Автоматично деплоїть (webhook)           │
└──────────────────┬──────────────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────────────┐
│  Ваш сайт → Оновлено! ✅                            │
└─────────────────────────────────────────────────────┘
```

### Перевірка статусу деплою:

- **Netlify Dashboard** → Вкладка **"Deploys"**
- Тут ви побачите:
  - ✅ Успішні деплої (зелений)
  - 🔄 В процесі (жовтий)
  - ❌ Помилки (червоний)

### Корисні функції Netlify:

1. **Deploy Previews** - попередній перегляд PR перед мержем
2. **Rollback** - швидке повернення до попередньої версії
3. **Analytics** - статистика відвідувань
4. **Forms** - вбудовані форми (альтернатива Supabase)
5. **Functions** - serverless функції

### Troubleshooting:

**Проблема: Білд падає**
- Перевірте логи в Netlify Deploy Log
- Переконайтеся що всі залежності встановлені

**Проблема: Сайт відкривається, але білий екран**
- Перевірте консоль браузера (F12)
- Можливо не налаштовані змінні оточення для Supabase

**Проблема: 404 при перезавантаженні сторінки**
- Файл `netlify.toml` має редірект, перевірте що він закоммічений

### Безпека:

🔒 **ВАЖЛИВО:** Ніколи не комітьте файл `.env` в GitHub!
- Змінні оточення додавайте тільки через Netlify Dashboard
- Файл `.env` вже додано в `.gitignore`

## Альтернатива: Vercel

Якщо захочете спробувати Vercel замість Netlify:

1. Зайдіть на https://vercel.com
2. Підключіть GitHub
3. Імпортуйте репозиторій
4. Vercel автоматично визначить Vite проєкт
5. Deploy!

Vercel працює аналогічно до Netlify з автоматичним деплоєм.
